Fuel Up

@autoexpandOff true

Introduction

Time to refuel!

In this tutorial we'll add a fuel bar to your spaceship that depletes as you travel.

Make sure to catch the powerups to keep your ship from breaking down!

Step 1

üòµ The starter code is taking up a lot of room! Don't worry, the Arcade workspace will expand for you. Just scroll up and over (or down and over) to keep building.

‚ñ∫ Take a peek into the new ||statusbars:Status Bars|| category. You'll find ||variables(statusbars):set [statusbar] to create status bar sprite width [20] height [4] kind [Health]||. Drag one to the end of the ||loops:on start|| container.

‚ñ∫ To keep track of how much gas is left, set the argument for statusbar kind to Energy.

Tip: The ||statusbars:Status Bars|| category is an extension. To see what else you can do using extensions, open a game in your gallery, click ||statusbars:ÀÖ Advanced|| and choose ||extension:Extensions||.

let statusbar = statusbars.create(20, 4, StatusBarKind.Energy)

Step 2

If we want the status bar to show the details of mySprite, we'll need to link the two together.

‚ñ∫ Drop ||statusbars:attach [statusbar] to [mySprite] ‚äï|| into the end of the ||loops(noclick):on start|| container.

‚ñ∫ Click ‚äï on the new block to reveal options to change the position of the status bar in relation to mySprite. Can you figure out how to get the bar to show up below your ship?

let statusbar = statusbars.create(20, 4, StatusBarKind.Energy)
// @highlight
statusbar.attachToSprite(mySprite, -25, 0)

Step 3

‚è∞ The longer you're in the air, the more fuel you use ‚è∞

Here's how to make the fuel go down as time passes.

‚ñ∫ Drag an ||game:on game update every [500] ms|| container into the workspace. Adjust the time argument to 300 ms.

‚ñ∫ Drop a ||statusbars:change [statusbar] [value] by [0]|| block into the game update container.

‚ñ∫ Change the amount the status bar changes from 0 to -1.

Tip: Remember this step later. If the fuel runs out too fast in gameplay, you can come back and adjust these blocks.

let statusbar: StatusBarSprite = null
game.onUpdateInterval(300, function () {
    statusbar.value += -1
})

Step 4

‚õΩ Time to refuel ‚õΩ

You can drop gas canisters, energy crystals, or juicy hamburgers...whatever makes sense for the vessel you have.

The code for dropping fuel is a lot like the code for dropping enemies. For a refresher on how things work, find the myEnemy blocks in the workspace and use them as a guide.

‚ñ∫ Drag a new ||game:on game update every [500] ms|| container into the workspace and change the interval to 5 seconds (5000 ms).

‚ñ∫ Snap a ||variables(sprites):set [projectile2] to|| ||sprites(sprites):projectile [ ] from side with vx [50] vy [50]|| block inside the newest on game update container.

‚ñ∫ Click ||variables(noclick):[projectile2]|| and rename the sprite ||variables(noclick):[myFuel]||.

‚ñ∫ Click on the grey square to bring up the sprite editor so you can draw a fuel sprite (or choose one from the gallery.)

‚ñ∫ Play with the vx and vy arguments of the fuel until it's falling straight down at a decent speed.

Step 7

When your ship overlaps fuel, you'll want the gas to disappear as the tank refills.

‚ñ∫ Drag an ||sprites:on [sprite] of kind [Player] overlaps [othersprite] of kind [Player]|| container into the workspace.

‚ñ∫ Change the last argument from ||sprites:Player|| to ||sprites:Gas||.

‚ñ∫ To refill the status bar after grabbing fuel, snag a ||statusbars:set [statusbar] [value] to [0]|| block and snap it in to your newest overlaps container. Change the value from 0 to 100.

‚ñ∫ Finally, make sure the used fuel disappears by snapping a ||sprites:destroy [mySprite] ‚äï|| block into the bottom of the same overlaps container and replacing ||variables:mySprite|| with ||variables:otherSprite||

namespace SpriteKind {
    export const Gas = SpriteKind.create()
}

let statusbar: StatusBarSprite = null
sprites.onOverlap(SpriteKind.Player, SpriteKind.Gas, function (sprite, otherSprite) {
    statusbar.value = 100
    otherSprite.destroy()
})

Step 9

üåå If you run out of fuel, you'll be marooned in space! üåå

The threat is real.

‚ñ∫ To add consequences for an empty status bar, drag a ||statusbars:on status bar kind [Health] zero [status]|| container into the workspace.

‚ñ∫ Change the status bar kind to Energy.

‚ñ∫ Snap a ||game:game over <LOSE>|| block inside as the ultimate fate.

statusbars.onZero(StatusBarKind.Energy, function (status) {
    game.over(false)
})
